# Qortium Core v1.0.0-preview.15 Release Prep

Status: planned prerelease from `main` after the preview-network connectivity
fixes (UPnP, peer fan-out, reachable-advertisement, seed storage policy) and the
QDN render fixes merge.

Base release: `v1.0.0-preview.14`

Release target: `main` commit `8deb337fcf3a19e9597ff21b8a5551975aa49748`, which
includes the connectivity (#45) and QDN-render (#43, #44) changes.

Asset: `qortium-preview.zip`

## Purpose

This prerelease fixes preview-network peer connectivity and QDN data
availability. Nodes now open both their chain and QDN data ports via UPnP and
discover more peers instead of being limited to the two seeds, public-IP nodes
reliably advertise themselves as directly reachable, and the seed nodes store all
QDN data so they can serve and relay any resource. It also includes two QDN
render fixes, a theme-matched splash screen, and dependency bumps.

## Release Highlights

- Preview nodes now enable UPnP by default, mapping both the chain (24892) and
  QDN data (24894) ports on UPnP-capable routers. Previously UPnP was disabled,
  so a node behind NAT never opened an inbound port and could only reach the two
  seeds â€” leaving most nodes with about two peers and starving QDN fetches, which
  all had to relay through the seeds. (#45)
- `minOutboundPeers` raised from 4 to 8 for healthier peer fan-out once more
  nodes are reachable. (#45)
- A node with a configured external IP address now advertises itself as directly
  reachable even when UPnP is off and no inbound handshake has arrived recently,
  so public-IP seeds/VPS reliably serve and relay QDN data. This applies to both
  the chain and QDN data networks. (#45)
- The preview seed nodes now use `storagePolicy: ALL` so they store every QDN
  resource and can serve or relay it to peers. (#45)
- QDN render: a non-default identifier can now be used as a path segment. (#43)
- QDN render: rendering no longer clobbers a Q-App's display settings with
  hard-coded defaults. (#44)
- The splash screen now matches the system light/dark theme. (#42)
- Dependency bumps: gRPC 1.81.0 â†’ 1.82.0 (#40) and swagger-jaxrs2 2.2.50 â†’
  2.2.51 (#41).

Notes:
- This is a public preview prerelease. Preview accounts, data, and balances are
  not permanent.
- Existing preview nodes pick up the new `uPnPEnabled` and `minOutboundPeers`
  defaults automatically on their next update through Home, via the settings
  merge â€” unless an operator has locally overridden those keys, in which case
  their chosen value is preserved.
- The two preview seed nodes run separately (not through Home) and must be
  redeployed and restarted to apply `storagePolicy: ALL`.
- UPnP only helps nodes behind a UPnP-capable router; nodes behind CGNAT or
  non-UPnP routers still rely on relays through reachable nodes.
