# Qortium Core v1.0.0-preview.12 Release Prep

Status: planned prerelease from `main` after the block-archive distribution
and preview chain-config packaging changes merge.

Base release: `v1.0.0-preview.11`

Release target: the `main` commit that includes this release-prep document. Do
not tag earlier commits; the release should represent the full main history
including the block-archive distribution groundwork.

Asset: `qortium-preview.zip`

## Purpose

This prerelease packages the first serving-side block-archive distribution
groundwork, the local block online-account history needed by block explorers
and clients, and the preview chain-configuration packaging changes that let
future checkpoints ship without splitting the preview network.

## Release Highlights

- Nodes now store the stable reward-share identities that were online for each
  processed block, so the block online-accounts lookup can return meaningful
  historical data instead of depending on mutable minting-account indexes.
- The preview chain configuration is now bundled inside the release jar and
  loaded from there when available, while custom and local test configurations
  can still be loaded from external files.
- Chain-configuration fingerprints now ignore optional checkpoints, allowing
  trusted checkpoint data to be added or updated later without making updated
  nodes reject otherwise-compatible peers.
- Nodes that keep a block archive can now read archive chunks, build a
  reproducible archive manifest, and expose that manifest through
  `GET /blocks/archive/manifest`.
- Peers can advertise their archived height during handshake and answer
  archive-manifest and byte-range archive-chunk requests. This release only
  enables the serving side; automatic peer download and replay remains off.
- New tests cover the archive distribution messages, archive manifest
  generation, archive chunk reads, and the per-block online-account repository
  behavior.
