# Qortium Core v1.0.0-preview.8 Release Prep

Status: planned prerelease after PR #22 merges.

Base release: `v1.0.0-preview.7`

Release target: the final `main` commit produced by merging PR #22. Do not tag
the PR branch directly if the release is meant to represent the merged PR.

Asset: `qortium-preview.zip`

## Purpose

This prerelease packages the Core-side hardening and QDN app-rendering support
from PR #22. It is intended to give Qortium Home and preview testers a single
Core build that includes the API, QDN, Electrum, developer-proxy, and
auto-update safety changes reviewed in the June security-report pass.

## Release Highlights

- QDN render authorization now honors the rendered identifier, while broader
  service/name grants still cover all identifiers under that published service.
- QDN render context now includes text-size support alongside theme and
  language so apps can receive `_qdnTextSize` on launch.
- Electrum SSL connections now use normal certificate validation by default and
  support explicit SHA-256 certificate pins for trusted self-signed servers.
- Plaintext Electrum servers are ignored unless
  `allowPlaintextElectrumServers` is explicitly enabled.
- The developer proxy now requires `devProxyEnabled`, defaults away from
  `unsafe-eval`, and bounds rewritten HTML responses before buffering them.
- QDN chunked uploads are bounded by the maximum QDN file size, clean up partial
  temp files, return generic oversized-chunk errors, and reject overloaded chunk
  processing instead of running disk I/O on the submitting worker.
- Raw QDN downloads now validate byte ranges and downgrade browser-executable
  HTML/XHTML MIME types to `application/octet-stream`.
- API CORS responses now use a fixed allowed-header list instead of reflecting
  arbitrary preflight headers.
- Chain-parameter transaction builders now require the API key.
- Generated API SSL keystores no longer use the inherited static default
  password.

## Compatibility Notes

- Qortium Home does not need a matching Core API change for this release path.
  Home should continue loading apps through the Core render route and can pass
  text-size state to Core when it wants apps to receive `_qdnTextSize`.
- Operators using plaintext Electrum servers must either switch to SSL servers
  or set `allowPlaintextElectrumServers` deliberately.
- Operators using self-signed Electrum SSL servers should configure
  `certificateSha256Fingerprint` for those server entries.
- Local QDN app developers must set `devProxyEnabled` before starting the
  developer proxy. Apps that still require `unsafe-eval` during development
  need the separate `devProxyUnsafeEvalEnabled` opt-in.

## Post-Merge Release Checklist

Run these after PR #22 has merged and local `main` is updated to the merge
result:

```sh
git checkout main
git pull --rebase origin main
git fetch origin --tags

./preview/package-release.sh --output=target/qortium-preview-v1.0.0-preview.8.zip
cp target/qortium-preview-v1.0.0-preview.8.zip target/qortium-preview.zip
./preview/smoke-release-logging.sh --package=target/qortium-preview.zip --timeout=120

sha256sum target/qortium-preview.zip
unzip -p target/qortium-preview.zip qortium-preview/qortium.jar | sha256sum

git tag v1.0.0-preview.8 "$(git rev-parse HEAD)"
git push origin v1.0.0-preview.8
```

Then create the GitHub prerelease with `qortium-preview.zip` attached. Fill in
the final release body with the package SHA-256, packaged JAR SHA-256, and the
full merged commit SHA from `git rev-parse HEAD`.
