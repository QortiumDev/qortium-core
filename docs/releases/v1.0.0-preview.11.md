# Qortium Core v1.0.0-preview.11 Release Prep

Status: planned prerelease from `main` after the local-settings preservation
and name-free QDN preview endpoint changes merge.

Base release: `v1.0.0-preview.10`

Release target: the `main` commit that includes this release-prep document. Do
not tag earlier commits; the release should represent the full main history
including the preview launcher and QDN preview API changes.

Asset: `qortium-preview.zip`

## Purpose

This prerelease packages the preview launcher change that preserves local
operator settings across restarts and the new name-free QDN preview API used by
client apps to preview publish content before an account has a registered name.
It also includes a release-packaging fix so documented relative output paths
work when building preview zips.

## Release Highlights

- Preview launchers now merge runtime settings instead of overwriting them from
  the release template on every start, so local operator changes made through
  the settings API or by editing the runtime settings file survive normal
  restarts while untouched template values continue to follow future releases.
- Preview reset scripts still return the runtime to a clean template state, and
  the release docs explain how the template snapshot protects local settings on
  Linux/macOS and Windows.
- Core now exposes `POST /arbitrary/preview/{service}`, allowing clients to
  generate temporary previews for any QDN service from a local file or
  directory path without requiring a registered name, chain sync, clock
  accuracy, or an on-chain publish attempt.
- QDN preview failures now report an error instead of a success message when a
  preview cannot be generated, giving client apps a clearer path to show users
  what went wrong.
- Local assistant notes and generated update-package artifacts remain ignored
  by git, keeping normal working-tree status focused on source, docs, tests,
  and release files.
- The preview package builder now resolves relative `--output` paths before it
  changes into the staging directory, so commands like
  `--output=target/qortium-preview-v1.0.0-preview.11.zip` create the zip in the
  expected repository location.
