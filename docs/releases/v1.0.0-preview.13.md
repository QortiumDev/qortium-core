# Qortium Core v1.0.0-preview.13 Release Prep

Status: planned prerelease from `main` after the upload-based QDN preview,
worker/WebAssembly render policy, and group-transaction JAXB discriminator
changes merge.

Base release: `v1.0.0-preview.12`

Release target: the `main` commit that includes this release-prep document. Do
not tag earlier commits; the release should represent the full main history
including the upload-based preview and render-policy changes.

Asset: `qortium-preview.zip`

## Purpose

This prerelease packages the upload-based QDN preview endpoint that lets remote
and mobile clients preview local content without sharing the node's machine, the
relaxed content-security-policy that lets QDN-rendered apps run workers and
WebAssembly, and the JAXB discriminator fix that restores the group-transaction
build endpoints.

## Release Highlights

- A new upload-based QDN preview endpoint accepts content in the request body
  (base64, as a single file or a ZIP of a folder) and builds the same temporary
  unpublished `/render` preview the existing path-based endpoint returns, so
  local-content preview now works from the mobile app and from a desktop app
  pointed at a remote node. The upload is size-bounded, ZIP extraction is
  path-traversal protected, a single HTML file for the website service is
  wrapped as `index.html`, the staged content is deleted after the preview is
  built, the call requires the node's API key, and nothing is signed or
  published.
- QDN-rendered apps can now run in-worker code and WebAssembly: JavaScript and
  worker assets are served with a content-security-policy that permits in-worker
  execution and WebAssembly and that allows spawning workers from the app's own
  origin or from blob URLs, while all other (non-script) assets stay locked to
  their own origin as before. The local developer proxy was updated to match.
  This unblocks engines such as EmulatorJS/Emscripten that previously failed to
  start even after their data had downloaded.
- The group-transaction data classes now carry the missing JAXB
  `@XmlDiscriminatorValue` annotations, fixing a 500 error returned by the group
  transaction build endpoints.
