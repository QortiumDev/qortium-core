# Qortium Core v1.0.0-preview.9 Release Prep

Status: planned prerelease from `main` after the minting-group API merge.

Base release: `v1.0.0-preview.8`

Release target: the `main` commit that includes this release-prep document. Do
not tag earlier commits; the release should represent the full main history
including the minting-group lookup API.

Asset: `qortium-preview.zip`

## Purpose

This prerelease packages the minting-group lookup API used by Qortium Home and
the chat app's start-minting flow, the completed Core translation pass across
all twenty Home languages, and the remaining Electrum TLS and developer-proxy
hardening from the June security follow-ups.

## Release Highlights

- Group API responses now report whether each group is one of the chain's
  configured minting groups at the current height, across the group list,
  search, single-group, by-owner, and by-member endpoints, so apps can offer
  minting actions only where they apply.
- Core message catalogs (API errors, transaction validation results, and the
  system tray menu) are now translated for all twenty Home languages, with
  full locale-tag lookups, English fallback for incomplete bundles, and a CI
  test that enforces translation bundle completeness.
- Electrum SSL connections now support explicit trust modes with first-use
  SHA-256 certificate pinning, the bundled Electrum server list was refreshed
  with TLS pins, and ElectrumX chain height responses are corroborated across
  servers before being trusted.
- The developer proxy now streams responses with bounded buffering and
  constrains proxied asset content types.
